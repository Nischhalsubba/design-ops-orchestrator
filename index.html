Enterprise Grade Design System Workflow

The DesignOps Orchestrator.

Welcome to the definitive front-end workflow architected by Nischhal Raj Subba. This system is designed to eliminate the gap between Figma and Production Code. It utilizes the full power of the Gulp 5.0 ecosystem to automate performance, accessibility, and consistency.

Prerequisites

Node.js LTS

Required for Gulp 5.0 and ES Module support. Version 18+ is mandatory.

Gulp CLI

Install globally to run commands: npm install --global gulp-cli

Figma API Key

To utilize the sync:tokens task, you need a Personal Access Token.

The "World-Class" Structure

We adhere to the 7-1 Sass Pattern and Atomic Design principles. This structure is scalable from small microsites to massive enterprise dashboards. 

root/
├── .github/              # CI/CD Workflows
├── gulp/                 # Automation Logic
│   ├── config.js         # Paths & Plugin Configs
│   └── tasks/            # Split tasks (ESM)
├── src/
│   ├── assets/           # Static Files
│   │   ├── fonts/
│   │   ├── icons/        # Raw SVGs for Sprites
│   │   └── img/          # Raw Images (Source)
│   ├── data/             # JSON Content for Pug
│   ├── markup/
│   │   ├── layouts/      # Master Templates
│   │   ├── components/   # Mixins & Partials
│   │   └── pages/        # Route Views
│   ├── scripts/          # TypeScript / ESNext
│   └── styles/           # The 7-1 Sass Pattern
│       ├── abstracts/    # _variables.scss, _mixins.scss
│       ├── base/         # _reset.scss, _typography.scss
│       ├── components/   # _buttons.scss, _cards.scss
│       ├── layout/       # _grid.scss, _header.scss
│       ├── pages/        # Page-specific overrides
│       ├── themes/       # Dark/Light mode tokens
│       ├── vendors/      # 3rd Party CSS
│       └── main.scss     # Import Hub
├── dist/                 # Production Artifact
└── gulpfile.js

Why this structure?

  • Separation of Concerns: Data, Markup, and Logic never mix.
  • Scalability: Adding a new component only requires adding files to the specific `components/` folders.
  • Automation Friendly: The `gulp/` folder allows us to change build tools without touching the source code.

The Data Layer

We utilize gulp-data to inject JSON into Pug templates. This allows you to build data-driven components (like product grids) without a backend.

The Plugin Ecosystem

We don't just use "standard" plugins. We use the specialized tools that 1% of developers know about to achieve 100% scores on Lighthouse.

1. HTML & Accessibility

Plugin Description Why it's essential
gulp-pug Template engine with robust logic. Cleaner syntax, supports mixins and includes.
gulp-w3c-html-validator Validates HTML against W3C standards. Ensures semantic correctness during build.
gulp-a11y Accessibility auditing. Checks output against WCAG 2.1 standards.
gulp-sitemap Generates sitemap.xml. Critical for SEO in production builds.
gulp-htmlmin Minifies HTML. Reduces payload size by removing whitespace/comments.

2. Styles & Internationalization

Plugin Description Why it's essential
gulp-sass Dart Sass compiler. Industry standard for modular CSS.
gulp-postcss CSS Transformations. Enables Autoprefixer, Tailwind, and future CSS syntax.
gulp-purgecss Removes unused CSS. Drastically reduces file size by analyzing HTML usage.
gulp-rtlcss LTR to RTL conversion. Automatically generates Arabic/Hebrew stylesheets.
gulp-group-css-media-queries Optimizes media queries. Combines split media queries into single blocks.
gulp-critical Critical path CSS. Inlines above-the-fold CSS for instant rendering.

3. Scripts & TypeScript

Plugin Description Why it's essential
gulp-esbuild Ultra-fast bundler. 100x faster than Webpack/Babel. Supports TS out of the box.
gulp-typescript TypeScript Compiler. Used for type-checking before bundling.
gulp-terser JavaScript minifier. Advanced compression for ES6+ code.
gulp-eslint Linter. Enforces coding standards and prevents bugs.

4. Assets & Optimization

Plugin Description Why it's essential
gulp-avif AVIF Converter. Superior compression (better than WebP).
gulp-webp WebP Converter. Standard next-gen format fallback.
gulp-svg-sprite SVG Spritesheet generator. Combines icons into one HTTP request.
gulp-rev Revision hashing. Appends content hash (style-a5b3c.css) for cache busting.
gulp-sri-hash Subresource Integrity. Security feature to ensure CDN assets haven't been tampered with.
gulp-brotli Brotli Compression. Pre-compresses assets for servers (better than Gzip).

Workflow Cookbook

Recipe: Syncing Design Tokens

Don't copy hex codes. Sync them.

# Fetch latest tokens from Figma API
npm run sync:tokens

# Output:
# [Sync] colors.scss updated
# [Sync] typography.scss updated
# [Sync] spacing.scss updated

Recipe: Adding Icons

Place any SVG into src/assets/icons/. Gulp automatically compiles them into a symbol sprite.

// Usage in Pug
svg.icon
  use(xlink:href="assets/sprite.svg#icon-user")

Recipe: Production Deployment

This command triggers the full optimization suite: Minification, PurgeCSS, Image Compression, and Hashing.

npm run build -- --production
