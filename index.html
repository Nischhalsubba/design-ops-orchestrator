Why I Built This by Nischhal Raj Subba

I spent years watching designers and developers fight a losing war against Entropy. Designers update a color in Figma; Developers miss the memo. A content writer fixes a typo in Notion; it takes three days to deploy. An animator updates a Rive file; the implementation breaks.

This "Drift" kills products. It kills morale. It kills excellence.

I built the DesignOps Orchestrator to be the immutable source of truth. It is not just a build tool; it is a translator. It speaks "Figma" and outputs "SCSS". It speaks "Notion" and outputs "JSON". It speaks "After Effects" and outputs "Optimized Assets".

This is my vision for 2026: A world where the tool doesn't matter, only the intent. Export your intent, and let the Orchestrator handle the code.

02. The Engines

The Token Engine

Figma ‚Ä¢ Sketch ‚Ä¢ Zeplin ‚Ä¢ Tokens Studio

How It Works

  • Install Tokens Studio plugin in Figma.
  • Export your variables as tokens.json.
  • Drop file into src/tokens/.
  • Engine auto-generates _generated-tokens.scss.
// src/styles/abstracts/_generated-tokens.scss
$global-colors-primary: #6366f1;
$global-font-size-lg: 1.25rem;
// Ready to use in any SCSS file!

The Content Engine

Notion ‚Ä¢ Obsidian ‚Ä¢ Google Docs ‚Ä¢ Jira

How It Works

  • Write in Notion/Docs. Export as Markdown & CSV.
  • Drop .md files into src/ingest/content/.
  • Engine parses Frontmatter (YAML) + Body.
  • Generates src/data/generated_content.json.
// src/data/generated_content.json
{
  "articles": [
    { "title": "DesignOps 101", "slug": "design-ops" }
  ]
}

The Motion Engine

Rive ‚Ä¢ Lottie ‚Ä¢ After Effects ‚Ä¢ ProtoPie

How It Works

  • Export .json (Lottie) or .riv (Rive).
  • Place in src/assets/animation/.
  • Engine minifies JSON and compresses binaries.
  • NEW: Generates _motion-manifest.json so JS knows exactly which files exist.
// dist/assets/animation/_motion-manifest.json
[
  "loader.riv",
  "success-confetti.json"
]
